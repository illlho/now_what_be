**용도**: 다른 환경에서 3단계부터 이어서 진행하기 위한 컨텍스트 문서

---

## 📌 AI에게 전달할 프롬프트

다른 컴퓨터에서 새로운 대화를 시작할 때, 아래 내용을 복사해서 AI에게 전달하세요:

```
@0_document/01_planning_orchestration_guide.md 와 @0_document/02_planning_agent_food_search_20260107.md 를 기반으로 AI Agent Service 기획을 진행 중입니다.

현재 1-2단계가 완료되었고, 3단계부터 이어서 진행하려고 합니다.

진행 방식:
- 질문은 하나씩 순차적으로 해주세요
- 다음 질문으로 넘어가달라는 얘기가 있을 때만 넘어가세요
- 그 전에는 현재 질문을 깊이 있게 디벨롭합니다
- 각 단계가 완료되면 문서를 업데이트하고 커밋해주세요

3단계부터 시작해주세요.
```

---

## 📋 현재까지 진행 상황 요약

### 프로젝트 개요
**서비스명**: 맛집 정보 수집 및 분석 AI Agent Service

**핵심 문제**:
- 5개 장소 비교 시 25개+ 블로그를 1시간 이상 수동 분석
- 핵심 정보 추출, 장단점 정리, 체험단 구분의 어려움

**타겟**: 맛집 탐방 좋아하는 20-30대 (주 1회+, 결과 활용률 높음)

### 1단계 완료 내용

**Agent 필요성**:
- 광고/체험단 구분 필요
- 객관/주관 정보 구분 필요
- 정보 충분성 판단 필요
- 특색 파악 및 비교 자동화 필요

**6가지 AI 판단 도출**:
1. 정보 충분성 판단
2. 객관/주관 구분
3. 모순 해결
4. 장소 특색 판단
5. 비교 기준 선택
6. 예외 상황 처리

### 2단계 완료 내용

**MVP 핵심 의사결정 3가지** (나머지는 향후 적용):
1. **정보 충분성 판단**
   - 충분: 전체 항목 3개 이상
   - 부족: 3개 미만 → 현재 데이터로 진행

2. **장소 특색 판단**
   - 언급 비중 가장 높은 항목
   - 동률이면 모두 특색

3. **비교 점수화**
   - 5개 항목: 맛(40%), 가격(20%), 서비스(20%), 양(10%), 웨이팅(10%)
   - 각 항목: 500점 시작, 긍정 +100, 부정 -100, 범위 0~1000
   - 종합 점수: /1000점

**정보 항목**:
- 필수: 주소, 메뉴
- 비교: 맛, 접근성 (접근성은 평가 항목에서 제외됨)
- 권장: 웨이팅, 주차, 영업시간

**MVP 제외**:
- 가격 정보 (OCR 필요)
- 개인화
- 객관/주관 구분 자동화
- 모순 해결 자동화
- 예외 상황 자동 처리

**성공 지표**:
- 정확도: 70-80%
- 시간: 10분 이내
- 비용: 장소당 500원 이하

## 🎯 3단계 시작을 위한 준비

### 3단계 목표
**Agent 상태 및 판단 로직 설계 (20분)**

주요 작업:
1. 상태(State) 정의
2. 판단 기준표 작성
3. Agent 플로우 설계
4. 순환 구조도 작성

### 3단계 첫 질문 (예상)
AI가 물어볼 것으로 예상되는 첫 질문:

"Agent가 내부적으로 유지해야 하는 핵심 상태는 무엇인가요?"
- 예: 시도 횟수, 수집된 정보, 신뢰도, 블로그 개수 등

### 일관성 유지를 위한 핵심 원칙

1. **질문 방식**:
   - 하나씩 순차적으로
   - 답변 후 깊이 파고들기
   - 명시적으로 "다음" 요청 시에만 넘어감

2. **답변 스타일**:
   - 구체적 예시 선호
   - 정량적 기준 제시
   - "예시 보여주세요" 자주 요청

3. **의사결정 패턴**:
   - 단순성 선호 (복잡도 배제)
   - "있는 정보로 최선" 철학
   - 완벽주의 배제
   - 비용/시간 통제 중시

4. **각 단계 완료 시**:
   - 저장 후 커밋 요청
   - 커밋 메시지는 구체적으로
