# 테스트 오케스트레이션 문서

## 테스트 유형별 참고 문서

| 테스트 유형 | 참고 문서 |
|------------|----------|
| LangGraph 워크플로우 | `test_langgraph_workflow.md` |
| 일반 API 엔드포인트 | 본 문서 기본 절차 |
| 통합 테스트 | 본 문서 기본 절차 |

## 테스트 원칙

### 파일 관리
**테스트 과정에서 생성했던 파일은 제거하고, 결과는 채팅으로만 안내.**
- 임시 파일(`test_*.json`, `test_*.py` 등) 테스트 후 즉시 삭제
- 결과는 파일 저장 없이 채팅으로만 보고
- 테스트 스크립트/설정 파일 프로젝트에 포함하지 않음

### 결과 보고
- 모든 결과 채팅으로 보고
- 결과 파일 생성하지 않음
- 이슈/개선사항 채팅으로 공유

### 환경 설정
- 테스트 전 필수 환경 변수 확인
- 환경 설정 파일 프로젝트에 포함하지 않음

## 테스트 절차

1. **사전 확인**: 서버 상태, 환경 설정, 패키지 설치, 참고 문서 확인
2. **테스트 실행**: 참고 문서에 따라 정상/에러/경계값 테스트
3. **결과 정리**: 채팅으로 요약, 임시 파일 삭제, 이슈 보고

## 기본 테스트 케이스

- **정상**: 기본 기능 동작, 다양한 입력 형식
- **에러**: 잘못된 입력, 필수 파라미터 누락, 환경 설정 오류

## 예외 사항

- 프로젝트 포함 테스트 코드(`tests/` 등)는 제외
- CI/CD 테스트 스크립트는 별도 관리
- 프로젝트 일부인 테스트 설정 파일은 예외
