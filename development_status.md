# 개발 현황 파악 및 다음 단계 추천

## 문서 목적

**분석 및 추천 전용. 이행 시 개발 진행 금지.**

## 실행 원칙

### 금지
- 코드 수정/생성
- 파일 추가/삭제
- 구현 작업

### 허용
- 코드베이스 분석
- 현황 파악/정리
- 점수 기반 추천
- 채팅으로 보고

## 분석 절차

### 1단계: 코드베이스 분석
- 파일 구조/의존성/데이터 흐름/상태 정의

### 2단계: 구현 상태 검증
- 워크플로우: 그래프, 노드, 엣지, 조건부 분기
- 노드: 입력/출력/처리/에러
- 유틸리티: 외부 API, 데이터 처리
- 상태: 정의, 스텝 로깅
- 에러: 노드/워크플로우 레벨

### 3단계: 미구현 항목 식별
- 미연결 노드/미정의 상태/미구현 유틸리티/미처리 에러

### 4단계: 점수 기반 추천
- 평가 기준별 점수 산정
- 종합 점수로 순위 결정

## 평가 기준

### 점수 체계 (1-5점)

| 기준 | 설명 | 가중치 |
|------|------|--------|
| 중요도 | 비즈니스 가치, 사용자 영향 | 30% |
| 의존성 | 선행 구현 필요 여부 (낮을수록 고점수) | 25% |
| 난이도 | 구현 복잡도 (낮을수록 고점수) | 20% |
| 효과 | 구현 후 개선 정도 | 15% |
| 독창성 | 서비스 차별화 기여 | 10% |

### 점수 기준

| 점수 | 중요도 | 의존성 | 난이도 | 효과 | 독창성 |
|------|--------|--------|--------|------|--------|
| 5 | 핵심/필수 | 의존 없음 | 1일 이내 | 대폭 개선 | 시장 차별화 |
| 3 | 보조 기능 | 2-3개 선행 | 1주일 | 측정 가능 | 업계 표준 |
| 1 | 선택적 | 복잡한 의존 | 1개월+ | 불명확 | 차별화 없음 |

### 종합 점수 계산

```
종합점수 = (중요도 x 0.30) + (의존성 x 0.25) + (난이도 x 0.20) + (효과 x 0.15) + (독창성 x 0.10)
```

## 분석 포인트

### 워크플로우 구조
- StateGraph 구성
- 노드 등록/연결
- 조건부 엣지
- 엔트리/종료 포인트

### 노드
- 입력 처리: 파싱, 검증, 상태 초기화
- 비즈니스 로직: LLM 호출, 데이터 처리
- 외부 API: 서비스 연동
- 결과 처리: 정렬, 필터링, 요약

### 상태
- WorkflowState 필드
- 노드 간 전달
- 스텝 기록

### 유틸리티
- LLM: 구조화 출력, 토큰 추적
- 외부 API: 호출, 에러 처리
- 데이터: 변환, 검증, 포맷팅

## 주의사항

### 분석 시
- 코드 검색으로 실제 구현 확인 (문서만 신뢰 금지)
- 의존성 체인 완전 파악
- 에러 전파 경로 추적

### 점수 산정 시
- 주관 배제, 코드 기반 평가
- 의존성 순서 반드시 고려
- 동점 시 의존성 낮은 항목 우선

### 결과 보고 시
- 추천만 제공, 구현 금지
- 순위 근거 명시
- 사용자 결정 대기
